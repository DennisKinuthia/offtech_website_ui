Git

A version control system that allows developers to manage source code changes and collaborate on software development projects. Git provides a way to track changes to code over time, including who made the changes, when they were made, and what was changed.

Continuous Integration (CI)

Regularly and automatically building, testing, and integrating code changes into a shared repository. The goal of CI is to detect and fix problems early in the development process, before they can cause issues in production.

Operations:

  • Unit Tests
  • User Stories
  • Git Operations
  • Scripts(yaml scripts)

Containerization

Lightweight, portable, and self-contained environments for running applications. Containers are isolated from the host operating system and other containers running on the same machine, providing a secure and consistent environment for running applications.

Monitoring

Observing and measuring the performance, availability, and behavior of systems, applications, and infrastructure components. Monitoring provides real-time visibility into the state of the system, allowing administrators to identify and resolve issues before they impact users.
